Core Loops

Two Operational Cycles

Ingestion handles new material. Self-update improves extraction behavior through bounded experimentation.

📥

Ingest Loop

Triggered when new material arrives (file, conversation, batch)

1
ingest-source
Normalize input, create source note with evidence blocks (anchored text chunks)
2
extract-knowledge
Extract candidate entities, concepts, claims, and relationships from evidence
3
resolve-identity
Match candidates against existing vault notes by title, alias, and slug
4
update-vault
Apply minimal patches — append sourced claims, add relationships, create new notes
5
repair-graph
Fix duplicates, orphans, broken wiki-links without changing source meaning
6
evaluate-graph
Score vault quality: hard constraint checks + soft metric calculations
🔄

Self-Update Loop

Scheduled daily — bounded autonomous improvement cycle

1
evaluate-graph
Run baseline on frozen + metamorphic + retrieval benchmark suites
2
cluster failures
Group failure patterns from benchmark results and threshold warnings
3
propose-skill-change
Generate one hypothesis targeting one skill file, with expected improvement
4
isolated evaluation
Apply candidate in copied workspace, re-run all benchmarks
5
accept / reject
Compare baseline vs. candidate against acceptance criteria
6
record outcome
Write artifacts, update pitfalls.md, append to ledger.jsonl

Design Principles

Safety Properties

Every design decision constrains what the system can do, not what it can learn.

📎

Source-Grounded

Every factual claim must cite a specific source block anchor. No provenance, no entry.

🛡️

Dry-Run Default

All ingestion is preview-only by default. Explicit --apply required to write to the vault.

♻️

Idempotent

Re-ingesting the same source produces a no-op plan. Identity is snapped by title, alias, and slug.

🤝

Conservative Merging

A false merge is worse than a missed merge. Low-confidence cases go to unresolved inbox.

🔒

Bounded Self-Update

Only skills/*.md files can change. Hard constraints can never be relaxed during auto-update.

🌐

Multi-Backend

Deterministic (zero-dep), OpenAI, and Anthropic extractors — all profile-driven and benchmarkable.

Quality Signals

Evaluation Framework

Hard constraints gate acceptance. Soft metrics guide improvement direction.

● Hard Constraints (must pass)

Source coverage for all claims GATE
Graph parseability GATE
Idempotent re-ingestion GATE
No silent contradiction flattening GATE
No unsupported destructive edits GATE

● Soft Metrics (guide direction)

citation_coverage TRACK
duplicate_note_rate TRACK
orphan_note_rate TRACK
isolated_note_rate TRACK
canonical_link_density TRACK
graph_churn TRACK
retrieval_pass_rate TRACK

⚖️ Self-Update Acceptance Criteria

✓ No new benchmark failures ≤ 0
✓ No new hard constraint issues ≤ 0
✓ Duplicate rate regression ≤ 2%
✓ Isolated note rate regression ≤ 3%
✓ Graph churn regression ≤ 3%
✓ Retrieval QA regression ≤ 3%
✓ Minimum score improvement ≥ 1%
✓ Primary failure must improve ≥ 1%
Quick Reference

Everyday Usage

All commands are thin, deterministic wrappers. Dry-run by default, explicit apply.

📥 Ingest a File

Preview extraction plan, then apply to write

# dry-run (preview plan) python3 main.py ingest-file \ --vault /path/to/vault \ --input doc.md # apply to vault python3 main.py ingest-file \ --vault /path/to/vault \ --input doc.md --apply

💬 Ingest a Conversation

Extract knowledge from chat logs

python3 main.py ingest-conversation \ --vault /path/to/vault \ --input chat.txt --apply

📚 Batch Ingest

Process a directory of files at once

python3 main.py ingest-batch-files \ --vault /path/to/vault \ --input-dir files/ \ --profile openai_primary

🩺 Check Vault Health

Run integrity checks and compute quality metrics

python3 main.py check --vault /path/to/vault python3 main.py metrics --vault /path/to/vault

🧪 Run Benchmarks

Execute frozen, metamorphic, or retrieval test suites

python3 main.py benchmark-run \ --manifest benchmarks/frozen/manifest.json python3 main.py benchmark-run \ --manifest benchmarks/metamorphic/manifest.json

🔄 Self-Update Cycle

Run one bounded improvement cycle

python3 main.py self-update-run \ --proposal-profile openai_primary \ --benchmark-profile openai_primary # auto-apply if accepted python3 main.py self-update-run \ --proposal-profile openai_primary \ --benchmark-profile openai_primary \ --apply-accepted

🔍 Retrieval QA

Test source-grounded question answering

python3 main.py qa-run \ --vault /path/to/vault \ --questions benchmarks/retrieval/questions.json

📋 Inspect History

View recent run records from the ledger

python3 main.py ledger tail --limit 5 python3 main.py diff \ --before snap_a.json \ --after snap_b.json
